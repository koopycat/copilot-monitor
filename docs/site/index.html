● Local-only · No telemetry · Single binary

Know exactly what GitHub Copilot is doing on your machine.

A local HTTP reverse proxy that records per-request metadata, token counts, latency, and an estimated AI-credit cost. Everything is stored in SQLite on your machine. No cloud. No telemetry. No prompts, completions, source code, or auth headers are ever written to disk. 

# run the proxy
./copilot-monitor run &

# start the dashboard API
./copilot-monitor serve
# → open http://127.0.0.1:7734
View on GitHub → Quickstart Architecture API reference

Why

GitHub Copilot picks models automatically, charges AI credits invisibly, and exposes usage summaries only on github.com with a 24-hour delay. You don't know which model handled your last prompt, whether you got a cache hit, or what that refactor session actually cost you.

Copilot Monitor gives you the raw numbers from your own machine: per-model token counts, latency, estimated cost, and 30-minute session groupings. All in a local dashboard with a period selector (today / yesterday / 7d / 30d / 90d / 365d) and a JSON API for your own scripts.

What you get

$

Real cost estimates

Per-model AI-credit list-price estimates, broken down by prompt, cached input, cache write, and output tokens. Clearly labeled as estimates — never your actual bill.

⌘

CLI + JSON API

stats, cost, today, sessions, and export for shell scripts. Every endpoint also serves JSON for your own tooling.

⏱

Live session view

Requests are grouped into sessions using a 30-minute inactivity gap. See the model mix, cost, and duration of the session you're in right now.

📅

Period drill-down

Toggle between today, yesterday, 7d, 30d, 90d, and 365d in the dashboard. Every section — model distribution, sessions, cost — filters to the same window.

🔒

Local-only by design

Binds to 127.0.0.1. SQLite lives at ~/.local/share/copilot-monitor/store.db. No external requests. No analytics. No background uploads.

✓

Inspectable

Single Go binary. The schema is internal/store/schema.sql. Read the source, audit the captures, trust what you see.

What is and isn't stored

✓ Stored

  • Timestamp, endpoint, HTTP status
  • Model name (when known)
  • Latency (ms)
  • Token counts: input, cached input, cache write, output
  • Project label (optional)
  • Session grouping (derived)

✗ Never stored

  • Prompt text
  • Completion text
  • Source code or repository paths
  • Auth headers, cookies, API keys
  • Anything from request or response bodies

See SPEC.md for the full privacy contract.

Who it's for

Daily Copilot users

You use Copilot Pro or Business every day and want to know which models you're actually calling, and what your prompts cost.

Prompt optimizers

You're tuning prompts for cache hit rate. Copilot Monitor exposes cached-input vs. uncached tokens per session so you can see what works.

Privacy-sensitive devs

You don't trust cloud-side telemetry. Read the schema, inspect the source, verify nothing leaves 127.0.0.1.

Tooling authors

You want to pipe Copilot usage into your own dashboards. Every endpoint returns JSON; queries are SQL on a SQLite file you own.

Get started

Clone the repo and use the devenv shell for the expected Go toolchain:

# enter the dev shell
devenv shell

# build
just build

# run the proxy + dashboard
./copilot-monitor run &
./copilot-monitor serve

# in another terminal
./copilot-monitor configure-vscode  # paste into VSCode settings.json

Open http://127.0.0.1:7734 for the dashboard.
