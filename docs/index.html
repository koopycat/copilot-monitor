Why it exists

AI coding tools burn through credits and tokens without showing you the bill. GitHub Copilot has zero local cost observability — you can't see what you're spending, on which models, until the monthly invoice arrives.

Copilot Monitor gives you the raw numbers from your own machine: per-model token counts, latency, estimated cost, and 30-minute session groupings. Local dashboard with a period selector (today / yesterday / 7d / 30d / 90d / 365d) and a JSON API for your own scripts.

How it works

A reverse proxy sits between your tools and LLM APIs, capturing metadata on the way through. Output goes to a local SQLite file and a dashboard on loopback.

request → copilot-monitor → response  ·  metadata → SQLite  ·  dashboard reads SQLite

What is and isn't stored

The privacy contract is enforced by the schema, not by policy. No prompt, completion, source code, or auth material is ever written to disk.

Recorded on the record

  • Timestamp, endpoint, HTTP status
  • Model name (when known)
  • Latency (ms)
  • Token counts: input, cached, cache write, output
  • Project label (optional)
  • Session grouping (derived)
Redacted

Never stored blacked out

  • prompt text
  • completion text
  • source code / repo paths
  • auth headers, cookies, keys
  • request / response bodies
  • Loopback only. The proxy and dashboard both bind to 127.0.0.1. Nothing is reachable from another machine.
  • No external network calls. The binary never phones home, never updates itself, and never sends analytics. Verified by reading the source.
  • Local file only. SQLite lives at ~/.local/share/copilot-monitor/store.db on your machine. Delete the file, every trace is gone.

Build from source

Clone, build, run. The devenv shell provides the Go toolchain.

PROCEDURE · terminal · zsh/bash 
# clone and enter the dev shell
git clone https://github.com/koopycat/copilot-monitor
cd copilot-monitor
devenv shell

# build, then run the proxy + dashboard
just build
./copilot-monitor run --dashboard

# point VSCode at the proxy — add to settings.json:
{
  "github.copilot.advanced": {
    "debug.overrideCapiUrl": "http://127.0.0.1:7733/copilot"
  }
}

# reload VSCode, then use Copilot normally

Open http://127.0.0.1:7734 for the dashboard.
